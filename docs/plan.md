# Multi AI Workspace - 프로젝트 기획서 (plan.md)

## 1. 프로젝트 개요

**프로젝트명:** Multi AI Workspace
**유형:** 단일 페이지 웹 애플리케이션 (Vanilla JS)
**목표:** Claude, Gemini, ChatGPT 3개 AI를 하나의 화면에서 동시에 사용하고 통합 관리하는 워크스페이스 제공
**실행 방법:** index.html을 브라우저에서 직접 열어 사용 (서버 불필요)

---

## 2. 사용자 요구사항

### 2.1 핵심 사용자 시나리오

| 시나리오 | 설명 |
|---------|------|
| 동시 질문 | 하나의 질문을 3개 AI에 동시 전송하여 답변을 비교 |
| 선택 질문 | 특정 AI만 선택하여 질문 전송 |
| 대화 분류 | 프로젝트 폴더별로 대화를 정리 |
| 즐겨찾기 | 중요한 대화를 북마크하여 빠르게 접근 |
| 템플릿 활용 | 반복 사용하는 프롬프트를 저장/재사용 |
| 데이터 백업 | 대화 내용을 JSON으로 내보내기/가져오기 |

### 2.2 사용자 니즈 분석

- **효율성**: 탭 전환 없이 한 화면에서 3개 AI 비교
- **조직화**: 프로젝트/주제별 대화 분류 관리
- **재사용성**: 프롬프트 템플릿으로 반복 작업 간소화
- **보안**: API 키와 대화 데이터를 안전하게 로컬 관리
- **이동성**: USB 등으로 회사/집 어디서든 사용 가능

## 3. 기능 명세

### 3.1 통합 질문 시스템

- **입력창**: 하단 고정 입력 영역 (ChatGPT 스타일)
- **전체 전송**: 버튼 클릭 시 Claude, Gemini, ChatGPT 3개 AI에 동시 API 요청
- **개별 선택 전송**: 체크박스로 원하는 AI만 선택 후 전송
- **응답 표시**: 각 AI의 응답을 세로 카드 형식으로 독립 표시
  - 카드 상단: AI 이름 + 아이콘
  - 카드 하단: 평가 버튼
- **로딩 상태**: 각 AI별 독립적인 로딩 인디케이터

### 3.2 사이드바 (왼쪽 패널)

- **프로젝트 폴더**
  - 폴더 생성 / 삭제 / 이름 변경
  - 대화를 폴더로 드래그앤드롭 이동
- **일반 대화**: 폴더에 속하지 않은 대화 목록
- **즐겨찾기**: 북마크된 대화 필터 뷰
- **검색 기능**: 대화 제목 및 내용 검색
- **대화 관리**
  - 대화 제목 자동 생성 (첫 질문 기반)
  - 대화 삭제 / 이름 변경

### 3.3 프롬프트 템플릿

- 템플릿 생성 / 수정 / 삭제
- 카테고리별 분류 (개발, 번역, 작문 등)
- 변수 삽입 기능: `{주제}`, `{언어}` 등 플레이스홀더 지원
- 템플릿 선택 시 입력창에 자동 삽입

### 3.4 데이터 관리

- **저장소**: `localStorage` (대화, 폴더, 템플릿, API 키, 설정 모두 저장)
- **내보내기**: 전체/선택 대화를 JSON 파일로 다운로드
- **가져오기**: JSON 파일 업로드로 대화 복원
- **이동성**: JSON 내보내기/가져오기로 회사↔집 데이터 이동

### 3.5 설정

- API 키 입력/수정 (Claude, Gemini, ChatGPT)
- 테마 설정 (라이트/다크)
- 기본 AI 선택 설정

| 즐겨찾기     |  | 응답 내용...      |                      |
|   └ 대화 3  |  | [좋아요] [복사]   |                      |
|             |  +------------------+                      |

### 4.2 사이드바 (280px 고정)

- 상단: 새 대화 버튼
- 중단: 폴더 트리 + 대화 목록 (스크롤 가능)
- 하단: 내보내기/가져오기 버튼

### 4.3 메인 채팅 영역

- 상단: 현재 대화 제목
- 중단: 대화 히스토리 (질문 + AI 응답 카드들, 스크롤)
- 하단: 입력 영역 고정

## 5. 기술 스택

| 구분 | 기술 | 선택 이유 |
|------|------|----------|
| 플랫폼 | 단일 페이지 웹앱 (file:// 프로토콜) | 서버/빌드 불필요, 브라우저에서 바로 실행 |
| 언어 | Vanilla JavaScript | 경량, 의존성 없음 |
| 스타일 | CSS Grid + Flexbox | 전체 화면 반응형 레이아웃 |
| 저장 | localStorage | 브라우저 내장 저장소 |
| 통신 | Fetch API | 각 AI API 호출 |

## 6. 파일 구조

```
src/
├── index.html              # 메인 페이지 (엔트리 포인트)
├── app.js                  # 앱 부트스트랩
├── styles/
│   ├── main.css            # 전체 레이아웃, 공통 스타일, 테마
│   ├── sidebar.css         # 사이드바 스타일
│   └── chat.css            # 채팅 영역 스타일
├── scripts/
│   ├── api-handler.js      # AI API 통신 (Claude, Gemini, ChatGPT)
│   ├── storage.js          # localStorage 데이터 관리
│   ├── ui-controller.js    # UI 상태 관리, 이벤트 핸들링
│   └── folder-manager.js   # 폴더/대화 CRUD 관리
└── assets/
    └── icons/              # SVG 아이콘
```

## 7. 개발 단계 (Phase 2 로드맵)

### Step 1: 기본 UI 레이아웃
- `index.html` 기본 구조 (사이드바 + 메인 영역)
- `main.css` 전체 화면 Grid 레이아웃 + 다크/라이트 테마
- `sidebar.css` 사이드바 기본 스타일
- `chat.css` 채팅 영역 기본 스타일

### Step 2: 사이드바 폴더 관리 기능
- `folder-manager.js` 구현
- 폴더 CRUD (생성/읽기/수정/삭제)
- 대화 목록 렌더링
- 드래그앤드롭 이동

### Step 3: 채팅 입력/출력 UI
- `ui-controller.js` 구현
- 입력창 UI 및 전송 이벤트
- AI 응답 카드 렌더링
- 전체 전송 / 개별 선택 UI
- 평가 버튼

### Step 4: API 연동
- `api-handler.js` 구현
- Claude / Gemini / ChatGPT API 연동
- 로딩 인디케이터
- 에러 핸들링 UI

### Step 5: 데이터 저장/불러오기
- `storage.js` 구현
- 대화 내역 저장/불러오기
- 폴더 구조 저장/불러오기
- 설정값 저장/불러오기
- JSON 내보내기/가져오기

---

## 8. 보안 요구사항

- API 키는 `localStorage`에 저장, 코드 하드코딩 절대 금지
- 사용자 입력값 XSS 방지 (innerHTML 대신 textContent 사용)
- 대화 데이터는 로컬 저장소에만 보관, 외부 서버 전송 금지
- file:// 프로토콜에서 직접 실행 가능
